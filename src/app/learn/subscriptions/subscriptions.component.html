Subscriptions

When we talk about subscriptions, we are talking about a subscription to an observable stream. This means that we want updates to any new events/items that come through a stream. There are a few different ways that we can subscribe to subscriptions

Ways to subscribe

  • Subscribe method
    • Subscribe with .subscribe()
    • Returns back a subscription that must be unsubscribed from with .unsubscribe()
  • Async pipe
    • Subscribe in the html template: {{example}}
    • Observable is automatically unsubscribed from when the code block is no longer rendered

Avoid Nested Subscriptions

We never want to subscribe to an observable within another subscribe or operator. Example:
