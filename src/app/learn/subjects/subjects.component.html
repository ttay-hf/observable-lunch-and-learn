Subjects

According to rxjs, subjects are:
A Subject is like an Observable, but can multicast to many Observers. Subjects are like EventEmitters; they maintain a registry of many listeners.
Here's the basic usage of a subject
{{'const subject = new Subject<number>();'}}

{{'subject.asObservable().subscribe(val => console.log(val));'}}

{{'subject.next(1);'}}
{{'subject.next(2);'}}

{{'// this will log:'}}
{{'// 1'}}
{{'// 2'}}
Void Subjects
Note that subjects do not have to contain a value. You can use Subject<void> to be able to emit events without emitting an actual value. This is how our DestroyableComponent works.

Subject

Any subscriber will receive any values that are emitted after the subscription has been setup. If next() is called before the subscription, the subscriber will NOT receive that event

BehaviorSubject

Similar to subjects, but behavior subjects will emit its current value to a subscribe when the subscription is created. Since it will always emit a value, behavior subjects are instantiated with its starting value.
const behaviorSubject = new BehaviorSubject<number>(1);

ReplaySubject

Replay subjects are similar to behavior subjects, but they buffer X number of events and will replay the events in its buffer to a new subscriber.

AsyncSubject

Async subjects are a variant where the subscribers are sent the last value only after the subject is completed. Any events that are emitted before the last one are not sent to the subscribers.
This is similar to the last() operator.
